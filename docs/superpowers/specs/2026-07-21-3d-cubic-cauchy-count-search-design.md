# 3D Cubic Cauchy Sample-count Search Design

## Goal

Find the smallest reliably stable same-patch Cauchy stencil for the L-prism
Dirichlet normal-jump second-kind formulation without reducing accuracy or
asymptotic convergence order.

## Fixed algorithm

- Use strict `same_patch` Cauchy sampling during the count search.
- Keep the 3D harmonic Cauchy polynomial at degree three (16 coefficients).
- Keep the current third-order restrict route unchanged: tensor-product
  tricubic Cartesian interpolation and the joint two-sided cubic normal fit.
- Keep crossing ownership, native NURBS DOFs, spread, manufactured harmonic
  data, GMRES tolerance `2e-10`, restart, and iteration limits unchanged.
- Treat Neumann results as regression diagnostics, not as an optimization
  gate.

The surface Cauchy traces of a cubic 3D harmonic polynomial have 10 value
degrees of freedom and 6 normal-derivative degrees of freedom on a planar
face. `10/6` is therefore the theoretical minimum candidate, not an assumed
winner.

## Configuration

Add optional environment variables:

- `KFBIM_3D_CAUCHY_VALUE_COUNT`, default `48`;
- `KFBIM_3D_CAUCHY_NORMAL_COUNT`, default `28`.

Both must be positive, the normal count must not exceed the value count, and
the fitted system must pass the existing 16-column SVD rank check. Preserve
the positional command-line format. Non-default count runs write beneath the
selected policy directory in a `v<value>_n<normal>` child directory, so no
scan point overwrites another.

Print and record the Cauchy degree, restrict grid degree, restrict normal
degree, and requested/actual sample counts. The numerical restrict degree is
already three; this change makes that invariant explicit rather than changing
the formula.

## Search

### Stage 1: paired sweep

Run `N=16,32` for:

`10/6`, `12/8`, `16/10`, `20/12`, `24/16`, `32/20`, and the `48/28` control.

Record Dirichlet convergence, GMRES iterations and relative residual,
density/interior errors and order, exterior-normal residual, route mismatch,
runtime, stencil radius, and condition median/p95/maximum. Record Neumann
convergence and iterations separately.

### Stage 2: local cross sweep

Take the best two adjacent paired candidates from Stage 1. Cross their value
and normal counts, including the immediately neighboring listed count when it
is needed to determine whether value or normal sampling controls stability.
Do not run combinations below `10/6`.

### Stage 3: fine-grid verification

Run the best three distinct count pairs at `N=16,32,64`. Re-run `48/28` only
if its existing result is not already generated by the same executable and
configuration.

## Ranking and acceptance

Reject a pair if any Dirichlet solve does not converge, either density or
interior `Linf` error fails to decrease under either refinement, or the
`N=32` to `N=64` density or interior order is below two.

Rank accepted pairs lexicographically by:

1. smallest maximum Dirichlet GMRES count across all levels;
2. smallest positive iteration growth from `N=32` to `N=64` (decreasing
   counts are preferred equally);
3. smallest `N=64` density and interior errors;
4. smallest condition p95, stencil radius, and total sample count.

The final recommendation must report the full three-level result and compare
it with same-patch `48/28` and the original topological-nearest `48/28`
control. A low-iteration pair that loses second-order fine-grid convergence
is not accepted.

## Tests

Add parsing tests through application smoke commands: defaults must report
`48/28`, valid overrides must report the requested pair, and zero, negative,
non-numeric, or normal-greater-than-value inputs must fail with a clear
message. Existing selector and native-NURBS tests must continue to pass.
